Was ist git?

Eine dezentralisierte Versionsverwaltung.

Versionsverwaltung allgemein:

  • lokal SCCS, RCS
  • zentral SVN, TFVC, Perforce...
  • verteilt / dezentral git, Mercurial...
Wozu Versionsverwaltung wichtig ist sollte klar sein. Allgemein Versionsverwaltung besonders für Textdateien geeignet.
  • lokal: Änderungsindex in der Datei selbst
  • zentral: Ein Repository auf einem Server
  • dezentral: Ein Repository pro Entwickler
Vor- und Nachteile [CLICK]
  • lokal: nur an einem Arbeitsplatz möglich. Dadurch irrelevant für Softwareentwicklung im Team
  • zentral: Wenn das Hauprepository zerstört wurde schwierig wiederherzustellen (die history kann im schlimmsten fall komplett verloren gehen)
  • dezentral: Wenn das Hauprepository zerstört wurde kann es einfach wiederhergestellt werden

Vor- und Nachteile

Lock - Modify - Unlock

Copy - Modify - Merge

  • lock-modify-unlock: file locking kann bei der Zusammenarbeit hinderlich sein
  • copy-modify-merge: gut für zeilenbasierte Textdateien - schwierig bei binären Datein (Grafiken etc.)

Zusammenfassung

  • TFVC: zentral, Lock - Modify - Unlock oder Copy - Modify - Merge
  • GIT: dezentral, Copy - Modify - Merge

TFS hat ein repository, Lock - Modify - Unlock kann optional eingeschaltet werden - offline arbeiten schwierig

GIT hat ein repository, das komplette Repository muss gepullt werden (lokale Kopie) - offline arbeiten.

Entstehungsgeschichte Git

Entwicklung des Linux Kernels

  • zunächst "manuelle" Versionsverwaltung
  • Ab 2002 wurde Bitkeeper verwendet
  • 2005 Lizenzänderungen von Bitkeeper
  • Linus Torvalds begann mit der Entwicklung eines eigenen Versionsverwaltungssystems

Linux Open Source - Bitkeeper kommerziell

Fokus

  • Geschwindigkeit
  • Einfaches Design
  • Gute Unterstützung von nicht-linearer Entwicklung (tausende paralleler Branches, d.h. verschiedener Verzweigungen der Versionen)
  • Vollständig verteilt
  • Fähig, große Projekte wie den Linux Kernel effektiv zu verwalten (Geschwindigkeit und Datenumfang)

Git vs. TFVC

“Git is the default version control provider for new projects. You should use Git for version control in your projects unless you have a specific need for centralized version control features in TFVC. You can use TFVC repos with Git in the same Team Project so it's easy to add TFVC later if you need centralized version control.”
https://www.visualstudio.com/en-us/docs/tfvc/comparison-git-tfvc

Git vs. TFVC

Sehr ähnliche Features

Hauptvorteile TFVC

Besser um sehr viele oder sehr große Dateien zu verwalten

Einfacher Einstieg

Hauptvorteile git

  • vollständig offlinefähig (lokale History)
  • "automatische Backups"
  • kostenlos und open source
  • Branching
  • auch TFVC teilweise offlinefähig, aber dann keine branches etc.
  • nicht an repository gebunden
  • staging area - partielle commits

Fazit

git bietet alle Funktionalitäten von TFVC und darüber hinaus noch weitere
